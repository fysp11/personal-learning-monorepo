# Finom Workspace Instructions

## Purpose
This folder is the Finom interview workspace. Keep it easy to resume before each round without reconstructing context from transcripts.

## Primary Re-entry Points
- `README.md` is the top-level dashboard. Keep current stage, next action, strongest prep assets, and role/company context here.
- `application/` holds the saved role context and fit analysis.
- `interviews/<round>/README.md` is the round-specific entry point for prep, summary, and follow-up questions.
- `prep/` holds short tactical notes for the next interview.

## File Placement Rules
- Put evergreen company and role context in `README.md`.
- Put round-specific raw assets and summaries in `interviews/<round>/`.
- Put focused prep notes in `prep/`.
- Put saved application materials in `application/`.
- Put reusable source packs or packaged study material in `artifacts/`.
- Put interviewer-specific notes in `interviewers/`.

## Naming Rules
- Keep the company folder lowercase: `finom`.
- Keep interview folders numbered: `1-...`, `2-...`, `3-...`.
- Use readable slugs after the round number, such as `2-central-ai--ivo`.
- Prefer Markdown for synthesized notes.
- Keep raw source filenames unchanged for audio or transcript files.

## After Each Interview
- Save raw audio and raw transcript inside the relevant `interviews/<round>/` folder.
- Add or update `interviews/<round>/README.md` with interview purpose, strongest signals, open questions, and prep for the next step.
- Add or update cleaned summary files in that same folder using the existing naming style.
- Update the top-level `README.md` timeline and current status so the next action is obvious.
- If a new interviewer becomes important, create or update an `interviewers/<Name>.md` note rather than burying it in a transcript summary.

## Working Preference
- Keep the top-level README as the fastest one-minute re-entry document.
- Do not leave stale cleanup notes in the dashboard once the issue is resolved.
- If a transcript is partial or noisy, say that clearly and point to the cleaned summary that should be used instead.
