# Delphyr Workspace Instructions

## Purpose
This folder is the Delphyr interview workspace. Keep it usable as a fast re-entry point for future sessions.

## Primary Re-entry Points
- `README.md` is the top-level dashboard. Keep company context, current stage, next action, and links to the highest-signal prep files here.
- `INTERVIEW-PREP.md` is the evergreen Delphyr technical prep note.
- `interviews/<round>/README.md` is the round-specific prep and outcome file for that interview.

## File Placement Rules
- Put evergreen company context, timeline, and top links in `README.md`.
- Put round-specific prep, notes, and post-interview outcomes in `interviews/<round>/`.
- Put tactical study notes in `prep/`.
- Put reusable deliverables or packaged study material in `artifacts/`.
- Put deep research imports or long-form background material in `reports/`.
- Put recent company/news changes in `updates/`.
- Keep technical rehearsal code and examples in `code/`.

## Naming Rules
- Keep the company folder lowercase: `delphyr`.
- Keep interview folders numbered: `1-...`, `2-...`, `3-...`.
- Use readable slugs after the round number, such as `2-technical--dejan-tim`.
- Prefer Markdown for synthesized notes.
- Keep raw source filenames unchanged for audio or transcripts.

## After Each Interview
- Save raw audio and raw transcript inside the relevant `interviews/<round>/` folder.
- Add or update that round folder's `README.md` with interview context, strongest signals, open questions, and next-step prep.
- Add or update a cleaned summary file when needed, using the existing folder naming style.
- Update the top-level `README.md` timeline and current status so the latest stage is visible immediately.
- If the interview changes the company understanding, update `interviewers/`, `prep/`, or `updates/` rather than overloading the top-level README.

## Working Preference
- Keep the top-level README concise and dashboard-like.
- Avoid duplicating long prep content in multiple files; link to the strongest existing note instead.
- When information is uncertain, state that clearly instead of guessing.
