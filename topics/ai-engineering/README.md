# AI Engineering Topic

This workspace is for structured learning and experimentation in practical AI engineering: productionizing AI tools, designing safe workflows, and understanding how models interact with systems and regulations.

## Structure

- `README.md` — high level topic goals, learning flow, and quick entry points.
- `AGENTS.md` — instructions for any agent working inside this topic folder (keep it minimal and focused on structure).
- `prep/` — active study notes, question banks, flashcards, or iteration prompts for current learning sprints.
- `resources/` — curated references, papers, blog posts, and canonical docs that support the learning path.
- `experiments/` — prototypes, scripts, configs, or notebooks developed while testing ideas.
- `artifacts/` — generated summaries, decks, or outputs (including NotebookLM artifacts) that result from the study so they can be shared or reused.

## Learning Flow

1. Capture a learning intent in `prep/` (e.g., “prompting reliability” or “model evaluation checklist”).
2. Link the strongest supporting references under `resources/` so you can quickly revisit the signal.
3. Keep experiments under `experiments/` with short logs about what you tried and what worked.
4. Package reusable deliverables or NotebookLM packs under `artifacts/` so you can re-run or share them later.

Roadmap ideas:
- Build a prompt reliability checklist with concrete examples.
- Recreate a safe inference pipeline in `experiments/` that includes validation and rollback steps.
- Use NotebookLM or local notes to generate a weekly “AI engineering health” brief in `artifacts/`.
